# 01-02 Plan: Diagnostico de ambiente y hardware NXT

## Goal

Dejar una rutina reproducible para revisar dependencias y ejecutar las pruebas base del Dia 1 sobre el brick, motores y sensores.

## Tasks

1. Implementar `main.py doctor` para revisar Python y dependencias clave.
2. Implementar `robot.py` con carga de perfil, conexion al brick y pruebas por puerto.
3. Definir un `config/hardware.example.json` editable por equipo.
4. Generar reportes de salida en consola, Markdown y JSON.

## Success checks

- El proyecto puede correr un `dry-run` sin hardware.
- La rutina live puede intentar conexion real y registrar fallos o lecturas por puerto.
- El resultado es suficiente para llenar el checklist y la bitacora del Dia 1.
